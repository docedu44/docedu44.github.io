Reading

شاخصه ها#

  1. تمامی reading ها دارای یک heading و sub heading می باشند که در واقع توضیح دهنده کل موضوع متن می باشند.
  2. دارای 3 passage می باشد که معمولا passage 1 بیشتر توصیفی ( descriptive ) می باشد، در مقابل passage 2 و passage 3 بحثی و استدلالی ( discursive & argument ) می باشند.
  3. معمولا این passage ها به صورت 13-14-13 می باشد

انواع سوال#

انواع سوالات reading:

  1. Paragraph information block question
  2. Match heading block question
  3. Short answer block question
  4. Flowchart completion
  5. Table completion

روش کلی حل سوالات#

  1. حتما مثبت و منقی بودن جمله مشخص شود.
  2. در passage 1 باید جواب ها را بدون تغییر بنویسیم.
  3. اولین و آخرین جمله هر پاراگراف خوانده شود.
  4. در heading ها باید توجه داشت که آن ها بر اساس مثال ساخنه نمی شوند، پس مثال ها را نمی خوانیم.
  5. در واقع subject ها بسیار مهم هستند و باید حواسمان به آن ها باشد.

Paragraph information#

  1. اغلب scan یی هستند و باید دنبال اطلاعات ریز ( fact ) بگردیم.( descriptive )
  2. بیشتر مواقع report هستند.
  3. اکثرا باید دنبال اسم صدا ها - صفات - صفات تفضیلی - نام ها و... بگردیم و کلا باید نشانه ها را پیدا کنیم.

Match heading#

  1. اغلب skim یی هستند و باید دنبال ایده و استدلال جمله باشیم ( opinion ) بگردیم.( argument - discursive )
  2. در واقع heading با متن در ارتباط مستقیم هستند پس باید منطق جمله را پیدا کرد.
  3. بهتر است در ابتدا کلمات کلیدی هر heading پیدا کنیم.

Short answer#

  1. اغلب scan یی هستند و باید دنبال اطلاعات ریز ( fact ) بگردیم.( descriptive )
  2. در 85% مواقع اسم است و خیلی کم پیش میاید که صفت باشد و تقریبا 1% احتمال دارد که فعل باشد.
  3. سوالات همیشه به ترتیب در متن هستند.
  4. همیشه باید سوال آسان را پیدا کنیم که بهتر است سوال وسطی باشد و بعد بر اساس آن در متن بقیه را پیدا میکنیم.
  5. اگر سوال موردی باشد، حتما بین ها ( شاید در 2 خط ) and وجود دارد.
  6. فقط دنبال نام و صفت بگرد.

Flowchart#

  1. اغلب scan یی هستند و باید دنبال اطلاعات ریز ( fact ) بگردیم.( descriptive )
  2. همیشه جواب ها در 1/3 متن می باشند، یعنی در 1/3 اول یا دوم یا سوم.
  3. حتما B/border دارد یعنی اکثرا cue یا همان hint دارد.
  4. کلمات اطراف جای خالی در صورت سوال، حتما مترادف با کلمه اصلی می باشد.
  5. حتما اسم است.
  6. دقیقا کلمه داخل متن نوشته شود.

Table#

  1. اغلب scan یی هستند و باید دنبال اطلاعات ریز ( fact ) بگردیم.( descriptive )
  2. همیشه جواب ها در 1/3 متن می باشند، یعنی در 1/3 اول یا دوم یا سوم.
  3. حتما B/border دارد یعنی اکثرا cue یا همان hint دارد.
  4. حتما اسم است.
  5. حتما باید pattern جدول را پیدا کنیم - مثل heading هر ستون
